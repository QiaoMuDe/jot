# AI 助手更多技能 — 翻译功能 Spec

## Why

为 AI 助手工具栏增加可扩展的 "更多技能" 机制，首个技能实现翻译功能。用户可在输入消息前通过工具栏激活翻译技能，指定翻译方向（中→英 / 英→中），此后发送的每条消息都会注入翻译系统提示词，AI 将按指定方向翻译用户输入。

## What Changes

- **新增** `.ai-chat-toolbar` 中的 "更多技能" 按钮（`#aiChatMoreSkillsBtn`），点击展开下拉菜单
- **新增** 技能下拉菜单，首项为 "翻译"，点击后在同一菜单内展开方向选择下拉框

  ```
  ┌─────────────────────┐
  │ 翻译 ▸              │  ← 一级菜单项
  │                     │
  │  ┌─────────────┐    │  ← 点击后展开的翻译方向选择
  │  │ ○ 翻译为中文 │    │     (默认选中)
  │  │ ○ 翻译为英文 │    │
  │  └─────────────┘    │
  └─────────────────────┘
  ```
- **新增** 技能状态指示条（`#aiChatSkillBar`），位于工具栏与输入框之间，显示已激活的技能 chip（含叉号取消）
- **新增** 全局技能状态管理对象 `activeSkills`，存储当前激活的技能列表及配置
- **修改** `startStreaming()` 函数，在构建 `messages` 时拼接技能相关的 system prompt
- **新增** 前端翻译技能配置：翻译方向（`to_chinese` / `to_english`），对应不同 system prompt

## Impact

- Affected specs: AI 对话交互
- Affected code:
  - `frontend/index.html` — 新增 "更多技能" 按钮、下拉菜单、技能指示条
  - `frontend/src/css/components/ai-chat.css` — 新增技能按钮、下拉菜单、指示条样式
  - `frontend/src/js/ai-chat.js` — 新增技能状态管理、事件绑定、system prompt 注入

## ADDED Requirements

### Requirement: 更多技能按钮与下拉菜单

系统 SHALL 在 AI 助手输入区工具栏右侧新增一个 "更多技能" 按钮。

#### Scenario: 点击展开技能菜单
- **WHEN** 用户点击 "更多技能" 按钮
- **THEN** 显示一个下拉菜单，列出可用技能（首项：翻译）
- **AND** 再次点击按钮或点击页面其他区域，菜单关闭

### Requirement: 翻译技能选择

系统 SHALL 在用户点击 "翻译" 技能后，在同一菜单内展开方向选择下拉框。

#### Scenario: 选择翻译方向
- **WHEN** 用户点击菜单中的 "翻译" 技能
- **THEN** 在 "翻译" 菜单项下方展开一个方向选择区，提供两个选项： "翻译为中文" 和 "翻译为英文"
- **AND** 默认选中 "翻译为中文"
- **AND** 用户点击任一选项后，技能被激活，整个菜单关闭

### Requirement: 技能激活指示

系统 SHALL 在输入区域上方显示已激活技能的 chip 指示器。

#### Scenario: 技能激活后显示 chip
- **WHEN** 用户选择翻译技能及方向后
- **THEN** 在工具栏下方、输入框上方显示一个 chip，内容为 "翻译 → 中文"（或 "翻译 → 英文"）
- **AND** chip 右上角有一个叉号按钮
- **WHEN** 用户点击叉号
- **THEN** 技能被取消激活，chip 消失

### Requirement: 翻译 system prompt 注入

系统 SHALL 在发送消息时，若翻译技能已激活，将翻译 system prompt 注入到消息列表头部。

#### Scenario: 技能激活时发送消息
- **WHEN** 翻译技能已激活（如 "翻译为英文"）
- **AND** 用户发送消息
- **THEN** 请求 API 时在 messages 数组头部插入 system prompt

#### Scenario: 翻译技能与笔记引用共存
- **WHEN** 翻译技能已激活
- **AND** 笔记引用也处于激活状态
- **THEN** 两个 system prompt 按顺序拼接：先笔记引用，后翻译指令

### Requirement: 技能状态持久化

系统 SHALL 将技能激活状态存储在内存中（当前会话有效），切换会话或刷新页面后重置。

## REMOVED Requirements

无

## Translation System Prompts

### 翻译为英文
```
# Role: 专业翻译助手

## Core Task
将用户发送的每条消息精准翻译成英文。

## Guidelines
- 准确传达原文含义、语气和风格，不增不减
- 遵循英文语法规范和地道表达，避免中式英语
- 专业术语保持行业通用译法
- 只输出翻译结果，不添加任何解释、备注或额外内容
- 如原文包含代码或专有名词（人名、地名、品牌名等），按英文惯例处理
```

### 翻译为中文
```
# Role: 专业翻译助手

## Core Task
将用户发送的每条消息精准翻译成中文。

## Guidelines
- 准确传达原文含义、语气和风格，不增不减
- 遵循中文语法规范和地道表达，避免翻译腔
- 专业术语保持行业通用译法
- 只输出翻译结果，不添加任何解释、备注或额外内容
- 如原文包含代码或专有名词（人名、地名、品牌名等），按中文惯例处理
```

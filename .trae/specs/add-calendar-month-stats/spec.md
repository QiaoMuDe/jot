# 日历月份统计摘要 Spec

## 原因

日历视图左侧栏的日历网格下方有约 300px 的空白区域，视觉上空荡荡。添加月份统计摘要可以填充空白，同时让用户一眼了解当月的笔记活跃度。

## 变更内容

- 在日历网格下方新增一个"本月统计"区域，展示：总笔记数、记笔记天数、最长连续记笔天数、活跃进度条
- 统计数据从已有的 `GetMonthNoteCounts` 返回值中计算，不需要新后端接口
- 翻月时统计自动跟随更新

## 影响范围

- 受影响的 spec：日历视图
- 受影响的代码：
  - `frontend/index.html` — 新增统计区域 HTML 结构
  - `frontend/src/css/components/calendar.css` — 新增统计区域样式
  - `frontend/src/js/calendar.js` — 新增统计计算和渲染逻辑

## 新增需求

### 需求：月份统计摘要

系统应在日历视图左侧栏的日历网格下方，显示当月的笔记统计信息。

#### 场景：正常月份
- **当** 日历视图加载或切换月份时
- **则** 在日历网格下方显示统计区域，包含：
  - **总笔记**：当月所有笔记的总数
  - **记天数**：当月有笔记的天数
  - **最长连续**：当月连续有笔记的最长天数
  - **活跃进度条**：`已记天数 / 已过天数` 的进度条

#### 场景：无笔记的月份
- **当** 当月没有任意笔记时
- **则** 统计数据显示为"总笔记 0 篇 · 记天数 0 天"，进度条为空

#### 场景：跨年跨月切换
- **当** 用户通过 prev/next 按钮切换到其他月份时
- **则** 统计区域自动更新为当月数据

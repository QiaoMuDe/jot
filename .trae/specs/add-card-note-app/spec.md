# Jot - 卡片笔记桌面应用 Spec

## Why
开发一款类似小米笔记的桌面端卡片式笔记软件，基于 Wails 框架，使用 Go + GORM + SQLite（纯 Go 驱动）作为后端，HTML/CSS/JS 作为前端，提供流畅的卡片笔记管理体验。

## What Changes
- 初始化 Wails v2 项目结构
- 使用 `github.com/glebarez/sqlite`（纯 Go SQLite 驱动，免 cgo）
- GORM ORM 操作 SQLite 数据库
- 前端使用纯 HTML/CSS/JS，卡片式 UI 设计
- 实现笔记的增删改查功能
- 支持笔记卡片网格/列表展示
- 支持笔记内容搜索
- 支持笔记分类/标签管理

## 布局结构
应用采用**左侧导航栏 + 右侧主内容区**的双栏布局：

```
┌──────────────┬──────────────────────────────────────┐
│  左侧导航栏    │                                      │
│              │                                      │
│  🔍 搜索模块  │        右侧主内容区                    │
│  [搜索框]     │                                      │
│              │    (卡片网格 / 搜索列表 / 设置页)       │
│  📋 笔记模块  │                                      │
│  (卡片列表)   │                                      │
│              │                                      │
│              │                                      │
│              │                                      │
│  ⚙️ 设置按钮  │                                      │
└──────────────┴──────────────────────────────────────┘
```

### 左侧导航栏（从上到下排列）
- **搜索模块**（顶部）: 类似百度搜索框，输入关键词搜索笔记，搜索结果展示在右侧，点击结果跳转到笔记详情
- **笔记模块**（中部）: 以卡片列表/缩略图形式展示笔记，点击切换右侧主内容区的详细展示
- **设置按钮**（底部）: 打开设置面板

## Impact
- 后端: Go 服务层、GORM 模型层、SQLite 数据库层
- 前端: 左侧导航栏（搜索模块 + 笔记模块 + 设置按钮）、右侧主内容区（卡片网格/详情/设置页）
- 配置: Wails 配置、数据库初始化

## Requirements

### Requirement: 数据库与模型层
系统 SHALL 使用 GORM + `github.com/glebarez/sqlite` 驱动管理 SQLite 数据库。

#### Scenario: 数据库初始化
- **WHEN** 应用启动
- **THEN** 自动创建/迁移数据库表（notes 表、tags 表、note_tags 关联表）

#### Scenario: Note 模型
- **WHEN** 系统定义 Note 模型
- **THEN** 包含字段：ID、标题、内容、颜色、置顶状态、创建时间、更新时间、删除时间（软删除）

#### Scenario: Tag 模型
- **WHEN** 系统定义 Tag 模型
- **THEN** 包含字段：ID、名称、颜色、创建时间

### Requirement: 左侧导航栏
系统 SHALL 提供左侧导航栏，分为搜索模块、笔记模块、设置按钮三个区域。

#### Scenario: 搜索模块（顶部）
- **WHEN** 用户在搜索框输入关键词并搜索
- **THEN** 右侧主内容区展示搜索结果列表，点击结果可跳转到笔记详情/编辑

#### Scenario: 笔记模块（中部）
- **WHEN** 用户点击笔记模块
- **THEN** 右侧主内容区展示卡片网格视图，左侧笔记模块区域显示笔记缩略卡片列表

#### Scenario: 设置按钮（底部）
- **WHEN** 用户点击设置按钮
- **THEN** 右侧主内容区展示设置页面

### Requirement: 笔记列表展示
系统 SHALL 在右侧主内容区以卡片网格形式展示笔记列表。

#### Scenario: 加载笔记列表
- **WHEN** 用户点击左侧笔记模块
- **THEN** 右侧以卡片网格形式展示所有未删除的笔记，按置顶状态和更新时间排序

#### Scenario: 空状态
- **WHEN** 没有笔记时
- **THEN** 显示友好的空状态提示

### Requirement: 笔记增删改
系统 SHALL 支持笔记的创建、编辑、删除（软删除）。

#### Scenario: 创建笔记
- **WHEN** 用户点击"新建"按钮
- **THEN** 打开笔记编辑器，保存后生成新卡片

#### Scenario: 编辑笔记
- **WHEN** 用户点击卡片上的编辑按钮或双击卡片
- **THEN** 打开笔记编辑器，加载已有内容

#### Scenario: 删除笔记
- **WHEN** 用户在卡片上执行删除操作
- **THEN** 软删除笔记（移入回收站），卡片从列表中移除

### Requirement: 笔记搜索
系统 SHALL 在左侧导航栏搜索模块提供关键词搜索。

#### Scenario: 搜索笔记
- **WHEN** 用户在左侧搜索框输入关键词并触发搜索
- **THEN** 右侧主内容区展示匹配的笔记结果列表，点击可跳转详情

### Requirement: 笔记置顶
系统 SHALL 支持笔记置顶功能。

#### Scenario: 置顶/取消置顶
- **WHEN** 用户切换笔记置顶状态
- **THEN** 置顶笔记优先展示在列表顶部

### Requirement: 标签管理
系统 SHALL 支持笔记标签分类。

#### Scenario: 管理标签
- **WHEN** 用户在设置页面创建/编辑/删除标签
- **THEN** 标签数据持久化存储

#### Scenario: 为笔记添加标签
- **WHEN** 用户编辑笔记时
- **THEN** 可为笔记关联一个或多个标签

### Requirement: 回收站
系统 SHALL 提供回收站功能。

#### Scenario: 查看回收站
- **WHEN** 用户进入回收站（设置在设置页面或笔记模块中）
- **THEN** 展示所有软删除的笔记

#### Scenario: 恢复笔记
- **WHEN** 用户在回收站中恢复笔记
- **THEN** 笔记恢复到正常列表

#### Scenario: 永久删除
- **WHEN** 用户在回收站中永久删除笔记
- **THEN** 笔记从数据库中彻底删除

### Requirement: 卡片外观定制
系统 SHALL 支持笔记卡片颜色定制。

#### Scenario: 更改卡片颜色
- **WHEN** 用户编辑笔记时选择颜色
- **THEN** 卡片背景色更新为所选颜色

### Requirement: 设置页面
系统 SHALL 提供设置页面，在右侧主内容区展示。

#### Scenario: 设置内容
- **WHEN** 用户点击左侧底部设置按钮
- **THEN** 展示设置页面，包含标签管理、回收站入口、主题设置等选项

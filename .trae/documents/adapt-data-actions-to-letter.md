# 数据操作区适配信笺风格 — 改造方案

## 现状分析

### 当前操作区结构

信笺下方的操作区包含 **5 个功能组、10 个操作行**：

```
┌──────────────────────────────────────────────────────────────────┐
│  ▌ 数据操作                     ← data-section-title (760px)     │
│                                                                  │
│  ┌────────────────────────────────────────────────────────────┐  │
│  │  📤 导出数据          备份为 .db 文件           ▸          │  │
│  │───────────────────────────────────────────────────────      │  │
│  │  📥 导入数据          从备份恢复               ▸          │  │
│  └────────────────────────────────────────────────────────────┘  │
│                                                                  │
│  ┌────────────────────────────────────────────────────────────┐  │
│  │  🗜️ 数据库瘦身        释放空间                ▸            │  │
│  │───────────────────────────────────────────────────────      │  │
│  │  📂 打开数据目录      查看数据库文件所在位置     ▸          │  │
│  └────────────────────────────────────────────────────────────┘  │
│                                                                  │
│  ┌────────────────────────────────────────────────────────────┐  │
│  │  🗑️ 清空 AI 会话      删除所有 AI 对话记录      ▸         │  │
│  └────────────────────────────────────────────────────────────┘  │
│                                                                  │
│  ┌────────────────────────────────────────────────────────────┐  │
│  │  ℹ️ 备份状态          暂无备份                             │  │
│  │───────────────────────────────────────────────────────      │  │
│  │  💾 一键备份          备份到自定义目录          ▸          │  │
│  │───────────────────────────────────────────────────────      │  │
│  │  📂 一键还原          从备份文件还原            ▸          │  │
│  └────────────────────────────────────────────────────────────┘  │
│                                                                  │
│  ──── 危险操作 ─────────────────────────────────────────────    │
│  ┌────────────────────────────────────────────────────────────┐  │
│  │  ⚠️ 恢复出厂设置    清空所有笔记、标签和设置    ▸         │  │
│  └────────────────────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────────────────────┘
```

当前特点：
- `.data-action-list` 宽度 = 760px（跟随 `.data-content-inner`）
- iOS 设置列表风格：圆角卡片、行式布局、图标+标签+描述+箭头
- `.data-section-title` 带左侧 accent 竖条装饰
- 信笺宽度 = 580px（比操作区窄）

### 设计冲突

| 元素 | 信笺 | 操作区 | 差异 |
|------|------|--------|------|
| 宽度 | 580px | 760px | 差 180px |
| 视觉基调 | 书信/叙事 | iOS 设置列表 | 功能 vs 拟物 |
| 交互性 | 纯展示 | 可点击操作 | 静态 vs 动态 |

## 三个方案

### 方案 A：最小适配（推荐）

只调整尺寸和间距，不改变操作区的 iOS 列表风格。

**改动范围：**
- 将 `.data-action-list` 的最大宽度从 760px 收窄至 **580px**（与信笺对齐）
- 移除 `.data-section-title`（"数据操作"标题），改为信笺内附言（见下）
- 将 "数据操作" 的 section 标题改为更柔和的形式

**优点：**
- 改动极小，风险低
- 视觉上对齐后整体更协调
- 操作功能保持清晰可辨

**缺点：**
- 风格上信笺和操作区仍有视觉差异（书信 vs 设置列表）

### 方案 B：附言风格

将操作区改造成信纸底部的「附言」（P.S.）区域，弱化卡片感。

**改动范围：**
- 移除 `.data-action-list` 的卡片边框和圆角背景
- 操作行改为极简文字链接样式（去掉箭头图标和背景 hover）
- 文字和信笺正文对齐（相同的左右内边距、字号）
- 分组之间用 `—` 短横线分隔而非卡片分隔
- 危险操作改用红色文字链接

**视觉预览：**
```
┌──────────────────────────────────────────────────────┐
│  ···················································  │
│                                                      │
│  —— Jot 数据助手                                     │
│                                                      │
│  ────                                                │
│                                                      │
│  导出数据  ·  导入数据                                │
│  数据库瘦身  ·  打开数据目录                          │
│  清空 AI 会话                                        │
│  备份状态：暂无备份  ·  一键备份  ·  一键还原          │
│                                                      │
│  ──── 危险操作 ────                                  │
│  恢复出厂设置                                         │
│                                                      │
└──────────────────────────────────────────────────────┘
```

**优点：**
- 风格高度统一，整页就像一封信
- 视觉干净、独特

**缺点：**
- 改动大，涉及 HTML/CSS/JS 三端
- 文字链接样式对"清空AI会话""恢复出厂设置"等危险操作不够醒目
- 长描述的可见性降低

### 方案 C：信纸附件卡片

保留操作区的 iOS 列表功能性，但将其视觉设计为「信纸附件」——像信纸底部粘贴了一张便签/附条。

**改动范围：**
- 移除 "数据操作" 的独立 section 标题
- 将 `.data-action-list` 的圆角从 `var(--radius-lg)` 改为更小的圆角
- 添加一个微妙的纸纹理或阴影，模拟"贴在信纸底部的便签"
- 添加一条波浪分隔线或剪刀线 `-·-·-·-` 分隔信笺和操作区
- 宽度对齐信笺的 580px
- 保持 iOS 列表的功能性（图标、箭头、hover 反馈）

**视觉预览：**
```
┌──────────────────────────────────────────────────────┐
│                                                      │
│  ...信纸正文...                                       │
│                                                      │
│                              —— Jot 数据助手          │
│                                                      │
│  ─·─·─·─·─·─·─·─·─·─·─·─·─·─·─·─·─·─·─·─·─        │
│                                                      │
│  ┌────────────────────────────────────────────────┐  │
│  │  📤 导出数据              备份为文件      ▸    │  │
│  │  📥 导入数据              从备份恢复      ▸    │  │
│  │  🗜️ 数据库瘦身           释放空间        ▸    │  │
│  │  📂 打开数据目录          查看位置        ▸    │  │
│  │  🗑️ 清空 AI 会话         删除对话        ▸    │  │
│  │  ℹ️ 备份状态              暂无备份              │  │
│  │  💾 一键备份              备份到目录      ▸    │  │
│  │  📂 一键还原              从文件还原      ▸    │  │
│  │  ⚠️ 恢复出厂设置          清空所有数据    ▸    │  │
│  │                                   (红色)       │  │
│  └────────────────────────────────────────────────┘  │
│                                                      │
└──────────────────────────────────────────────────────┘
```

**优点：**
- 保留现有功能交互（点击、hover、箭头），用户习惯不变
- 视觉上通过分隔线和统一宽度与信笺形成"主文+附件"关系
- 改动适中，主要是 CSS 调整

**缺点：**
- 功能较多时附件卡片较长（9 行操作）
- 比方案 A 改动稍大

## 方案对比

| 维度 | A 最小适配 | B 附言风格 | C 附件卡片 |
|------|-----------|-----------|-----------|
| 改动量 | 小（2 文件） | 大（3 文件） | 中（2 文件） |
| 风格统一度 | 中等 | 高 | 较高 |
| 交互保留 | 完整保留 | 丢失 hover/箭头 | 完整保留 |
| 危险操作醒目度 | 高（红色+卡片） | 低（文字链接） | 高（红色+卡片） |
| 风险 | 低 | 中 | 低 |

## 建议

推荐 **方案 A**（最小适配）作为首选，原因是：

1. 信笺是数据展示，操作区是功能入口——两者本质角色不同，强求风格一致反而降低可用性
2. iOS 设置列表本身就是极简风格，与 jot 整体设计语言匹配
3. 改动最小，只需调整宽度和标题样式
4. 如果后续觉得不够，可再升级到方案 C

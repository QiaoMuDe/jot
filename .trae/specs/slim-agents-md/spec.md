# AGENTS.md 瘦身计划 Spec

## Why

AGENTS.md 当前 435KB，内容庞杂。文件包含约 121+ 个章节，其中：
- **第 1-8 章**（~570 行）：项目静态分析（目录结构、模块、依赖、技术栈、设计模式等）—— **有长期价值，需保留**
- **第 9-121+ 章**（~1300+ 行）：按时间顺序累积的"新增记忆点"—— **本质是变更日志**，每新增一个功能/修复一个 bug 就往末尾追加

这些"新增记忆点"中：
- 大部分已过时（描述的是开发过程中的中间状态）
- 细节已在 `.trae/specs/` 中各有独立文档
- 部分章节后还有题号跳跃（如从十七直接跳到二十八）、重复编号（两个"三十"）

## What Changes

### 核心方案：保留骨架，清除脂肪

1. **保留并精简第 1-8 章**（目录结构、模块识别、依赖分析、设计模式、技术栈、补充分析、核心特点、待优化点）
   - 更新过时的数据（行数统计、文件数量等）
   - 删除冗余的细节描述
2. **完全移除第 9-121+ 章**（所有"关键记忆点"和"新增记忆点"）
3. **替换为**：
   - 一条简短说明：详细开发记录请查阅 `.trae/specs/` 各子目录
   - 保留最后 5-10 条最近的记忆点作为快速参考（可选）
4. **新增**：文件维护规范，说明什么时候需要更新 AGENTS.md

### 目标

- 文件大小从 435KB → 预计 50-80KB（~80% 瘦身）
- 保留项目架构概览价值
- 消除过时信息

## Impact

- 不影响任何运行时代码
- 仅修改 `AGENTS.md` 文档本身
- 需要确认用户是否仍需要保留某些特定的记忆点

## ADDED Requirements

### Requirement: AGENTS.md 精简

The system SHALL produce a slimmer AGENTS.md that retains only project-structural information.

#### Scenario: 瘦身后文件结构
- **WHEN** 用户打开 AGENTS.md
- **THEN** 文件应包含：项目概述、目录结构、核心模块、依赖关系、技术栈、设计模式、核心特点
- **AND** 不应包含按时间顺序累积的变更日志

#### Scenario: 历史记录可追溯
- **WHEN** 用户需要查找某个变更多细节
- **THEN** AGENTS.md 应指引用户到 `.trae/specs/<change-id>/` 查阅

## REMOVED Requirements

### Requirement: 所有"新增记忆点"章节（第 9-121+ 章）
**Reason**: 过时的变更日志，细节已分散在 `.trae/specs/` 各 spec 文档中
**Migration**: 用户在 AGENTS.md 中看到的指引将指向 `.trae/specs/`

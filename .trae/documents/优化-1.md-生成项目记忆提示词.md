# 优化 `1.md` 项目记忆生成提示词

## 当前状态分析

`1.md` 是一个用于生成项目记忆文件（AGENTS.md）的提示词模板，共 62 行，包含三部分：

* **一、核心分析维度**（6 大维度：目录结构、功能模块、依赖关系、设计模式、技术栈、补充分析）

* **二、输出要求**（6 条，最后一条是新加的维护规范）

* **三、配合要求**（说明用户会补充哪些信息）

## 需修改的三处

### 修改 1：指定章节编号格式

**位置**：`二、输出要求` 第 37 行附近

**原因**：当前没指定编号格式，不同 Agent 可能输出 `一、二、三` 或 `1. 2. 3.` 或 `Chapter 1`，风格不统一。且维护规范里的"第 1-9 章"与中文编号格式绑定。

**改动**：在第 37 行"保存到 `AGENTS.md` 文件中"后加一句说明

```markdown
报告章节使用中文数字编号（一、二、三…）
```

***

### 修改 2：区分"关键记忆点"与动态变更记录

**位置**：`二、输出要求` 第 40 行

**原因**：原文写"报告末尾需总结'项目核心特点''待优化点''关键记忆点'"，但"关键记忆点"这个措辞模糊——本意是**静态分析的关键结论**（如"该项目采用 MVC 架构"），与 Agent 后来追加的**动态变更记录**（changelog）混淆。结果导致 AGENTS.md 里既有"关键记忆点"又有"新增记忆点"，结构混乱。

**改动**：将第 40 行中的"关键记忆点"改为更明确的表述

```markdown
4. 报告末尾需总结"项目核心特点""待优化点"以及**初始静态分析的关键结论（非变更记录）**
```

***

### 修改 3：改为 Agent 自主执行，不等待用户输入

**位置**：整体替换 `三、配合要求` 第 54-61 行

**原因**：原文全篇是"请你先回复…我会补充…你接收后…"的被动等待模式。用户要求 Agent 直接自主探索项目，不需要任何等待和补充。

**改动**：替换为直接让 Agent 动手执行

```markdown
#### 三、执行方式

接收本提示词后，直接开始执行，无需等待用户补充信息：
1. 读取项目根目录及其子目录结构，梳理完整目录树
2. 识别关键代码文件，读取分析核心模块逻辑
3. 如缺少关键上下文（如业务场景说明），可根据目录结构和技术栈自行推断并标注"推断"
4. 按上述要求完成分析报告，保存到 `AGENTS.md`
5. 完成后告知"已完成项目记忆建立，后续可基于此回答项目相关问题"
```

***

# 修复"恢复出厂设置"遗漏项

## 问题分析

"恢复出厂设置"功能入口在数据管理模块 (`data-management.js`)，调用后端 `ResetDatabase()` 完成数据清理。

当前做法是增量式清理——逐个调用各 Service 的 ResetAll/DeleteAll。这种方式的**根本问题**是每次新增数据表都要手动追加清理逻辑，极易遗漏（已发现 AI 会话、AI 消息、API 预设 3 个遗漏项）。

**更优方案**：利用 SQLite + GORM 的 `Migrator().DropTable()` 直接删掉所有表，再走一遍 `AutoMigrate` + 初始化流程。未来新增任何表只要在 `AutoMigrate` 里注册过，重置就会自动处理，无需额外维护。

### 已发现的遗漏

| # | 遗漏内容                                                                | 影响            |
| - | ------------------------------------------------------------------- | ------------- |
| 1 | **AI 会话和消息** (`ai_sessions`/`ai_messages`)                          | 重置后旧对话仍可见     |
| 2 | **API 配置预设** (`api_profiles`)                                       | 重置后旧预设仍存在于列表  |
| 3 | **`reloadSettings()`** **缺失** **`loadNoteOpenFullscreenSetting()`** | 笔记打开全屏设置状态未刷新 |

***

## 变更计划

### 变更 1：后端 — 全部删表重建

**文件：`app.go`** **`ResetDatabase()`** **第 1438-1456 行**

将原本的 4 步增量清理替换为：删所有表 → AutoMigrate → 初始化默认数据。

```go
// ResetDatabase 清空所有数据，恢复出厂状态（删表重建）
func (a *App) ResetDatabase() error {
    // 1. 删除所有表（会自动处理外键依赖顺序）
    tables := []interface{}{
        &models.AIMessage{},
        &models.AISession{},
        &models.APIProfile{},
        &models.Setting{},
        &models.Note{},
        &models.Tag{},
        &models.Notebook{},
    }
    for _, table := range tables {
        if err := a.db.Migrator().DropTable(table); err != nil {
            return err
        }
    }

    // 2. 重新 AutoMigrate（等价于 InitDB 中的迁移）
    if err := a.db.AutoMigrate(&models.Note{}, &models.Tag{}, &models.Setting{},
        &models.Notebook{}, &models.AISession{}, &models.AIMessage{}, &models.APIProfile{}); err != nil {
        return err
    }

    // 3. 重新初始化默认标签
    if err := services.InitDefaultTags(a.db); err != nil {
        return err
    }

    // 4. 重建默认笔记本（重置后首次访问会自动创建）
    if err := a.notebookService.EnsureDefaultNotebook(); err != nil {
        return err
    }

    return nil
}
```

**为什么能一劳永逸**：`AutoMigrate` 里注册的模型清单与 `InitDB()` 保持同步。未来新增表只要加到 `InitDB` 的 `AutoMigrate` 中，重置时自然也会被清理和重建，无需再改 `ResetDatabase()`。

对应的 Service 清理方法（`noteService.ResetAll()`、`settingService.DeleteAll()`、`notebookService.ResetAll()`）不再需要被 `ResetDatabase` 调用，保留不动即可，不影响其他地方的引用。

### 变更 2：前端 — 补全 `reloadSettings()`

**文件：`frontend/src/js/data-management.js`** **第 31-39 行**

在 `reloadSettings()` 中追加 `loadNoteOpenFullscreenSetting()`：

```javascript
async function reloadSettings() {
    window.loadThemeSetting?.();
    window.loadFontSettings?.();
    window.loadSortSettings?.();
    window.loadPageSizeSetting?.();
    window.loadQuickNoteSetting?.();
    window.loadSyntaxHighlightSetting?.();
    window.loadCodeHighlightThemeSetting?.();
    window.loadNoteOpenFullscreenSetting?.();  // ← 新增
    window.loadAISettings?.();
}
```

# 信笺入场动画改造方案

## 总结

重新设计数据管理页面信笺的入场动画，从「弹性缩放」改为「信纸展开式」丝滑效果。

## 当前问题

当前动画 `letterReveal` 使用 `scaleY(0.3 → 1.025 → 0.99 → 1)` 配合强弹性缓动曲线 `cubic-bezier(0.34, 1.56, 0.64, 1)`，结果看起来像：**信纸被捏扁 - 弹开 - 上下抖几下**。用户感觉不到"展开"。

问题根因：

| 问题             | 原因                         |
| -------------- | -------------------------- |
| 不是展开是弹性        | 缩放是从中心均匀缩放的，真正展开应该从一侧开始    |
| 不像纸像橡胶         | 强弹性缓动(bounce curve)让运动有橡胶感 |
| 没有"从上到下展开"的序列感 | 所有内容同时出现                   |

## 新动画设计

### 核心效果：「信纸垂下展开」

视觉隐喻：信纸从上方缓缓放下，内容从上到下依次披露。

```
                    初始状态
   ┌────────────────────────────────┐   ← 顶部先出现(clip → 0)
   │                                │       ↑
   │                                │       │ clip-path: inset(0 0 100% 0)
   │                                │       │ 像幕布一样从上向下拉开
   │                                │       │
   │                                │       │ 同时信纸整体从下方 24px
   │                                │       │ 上升至最终位置
   └────────────────────────────────┘   ← 底部最后出现(clip → 100%)
```

### 技术方案：clip-path + transform 组合

使用 **`clip-path: inset()`** 实现从上到下的遮罩展开，配合轻量的垂直位移和弹性顿挫。

```css
@keyframes letterReveal {
  0% {
    opacity: 0;
    /* 完全被遮罩——什么都看不见 */
    clip-path: inset(0 0 100% 0);
    /* 信纸在下方 24px，稍微小一点 */
    transform: translateY(24px) scale(0.97);
  }
  25% {
    opacity: 1;
  }
  55% {
    /* 遮罩完全打开，内容全部可见 */
    clip-path: inset(0 0 0 0);
    /* 弹越过目标位，轻微的 overshoot */
    transform: translateY(-4px) scale(1.005);
  }
  80% {
    transform: translateY(1px) scale(0.998);
  }
  100% {
    opacity: 1;
    clip-path: inset(0 0 0 0);
    transform: translateY(0) scale(1);
  }
}
```

缓动曲线：`cubic-bezier(0.22, 1, 0.36, 1)` — 一种 **优雅的 ease-out**，开头迅速、结尾柔和，没有橡胶感。

时长：`0.9s` — 比现在略长，让展开过程可被眼睛捕捉。

### 为什么这是"展开"而不是"弹"

| 维度   | 之前           | 之后                   |
| ---- | ------------ | -------------------- |
| 视觉机制 | 整体缩放 + 弹跳    | top→bottom clip 遮罩展开 |
| 展开方向 | 无方向（从中心均匀缩放） | 从上到下，有明确的方向感         |
| 缓动   | 强弹性（橡胶感）     | 优雅 ease-out（纸感）      |
| 效果   | "被捏扁弹开"      | "信纸从上方垂放展开"          |

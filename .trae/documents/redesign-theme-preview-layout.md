# 主题预览 UI 重新设计

## 概述

保持现有的两个独立 section 布局不变（"主题设置" + "编辑器选项"），只重新设计系统主题和代码高亮主题的菜单+预览区的视觉样式和交互细节。代码预览支持纵向滚动。

## 当前问题

- **系统主题预览**：10 个扁平小色块排列简陋，像调试工具，不够直观
- **代码预览**：`overflow: hidden` 不可滚动，代码内容太少看不出高亮效果
- **整体视觉**：菜单和预览之间缺乏视觉连接，像是拼凑而非设计

## 改动方案

### 1. 系统主题预览 — 从色块改为"迷你 UI 卡片"

用一个等比例缩小的"虚拟 UI"卡片替代 10 个色块，展示该主题在实际界面中的样子：

```
┌──────────────────────────────┐  ← .theme-preview (border)
│  ┌─── 伪工具栏 ───────────┐  │
│  │ ●  笔记标题             │  │  ← --topbar-bg / --accent / --text-secondary
│  └─────────────────────────┘  │
│  ┌─── 伪卡片 ───────────────┐│
│  │ ████████████              ││  ← --card-bg, --text-primary (title bar)
│  │ ██████████████████████   ││  ← --text-secondary (body lines)
│  │ ████████                  ││
│  │ ┌────────┬─────────┐     ││
│  │ │  强调   │  次要    │     ││  ← --accent / --hover-bg buttons
│  │ └────────┴─────────┘     ││
│  └──────────────────────────┘│
│  ●  ●  ●  ●                 │  ← 语义色小圆点 (success/warning/error/accent)
└──────────────────────────────┘
```

所有着色通过 CSS `var()` 引用当前主题变量，切换主题时自动刷新，零 JS 开销。

### 2. 代码高亮预览 — 加滚动 + 视觉增强

- `max-height`: 120px → **200px**
- `overflow`: `hidden` → **`auto`**
- 添加自定义滚动条样式
- 预览代码加长（展示 import、function、hook、条件、console、JSX 等丰富语法节点）

### 3. 菜单增强 — 下拉菜单 + 预览之间的视觉连接

- 菜单选择器 `.theme-select-trigger` 视觉微调：选中状态增强
- 预览区轻微阴影过渡，切换主题时有层次感
- 系统主题预览和下拉菜单用卡片形式包裹在一起

## 文件改动

### 1. `index.html` — 替换系统主题预览内容

将 `#themePreview` 内的 `.theme-palette`（10 色块）替换为 `.mini-ui-preview`（迷你 UI 卡片）。代码预览容器不变。

### 2. `settings-panel.css` — 重写 Section N

替换旧色板样式为迷你 UI 卡片样式，更新代码预览样式，添加滚动条样式。

### 3. `main.js` — 更新 `buildCodePreview()`

- 预览代码加长
- CM6 theme: `maxHeight: '200px'`, `overflow: 'auto'`

## 验证

1. `npx vite build` 通过
2. 系统主题预览显示迷你 UI 卡片而非色块
3. 切换主题，迷你卡片颜色自动变化
4. 代码预览可纵向滚动，滚动条样式一致
5. 窗口缩放布局正常

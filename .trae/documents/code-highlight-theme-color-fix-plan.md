# 代码高亮主题配色修正计划

## 一、问题摘要

在 11 个 CM6 代码高亮主题中，有 3 个主题的部分 token 颜色设置过浅，导致在深色编辑器背景上肉眼难辨：

| 问题等级 | 主题 | 问题标记及色值 |
|:-------:|------|---------------|
| **严重** | **github-dark** | 字符串 `#A5D6FF`(粉蓝)、数值/属性 `#79C0FF`(浅蓝)、函数 `#D2A8FF`(浅紫) |
| **中等** | **one-dark-pro** | 运算符/meta/转义均使用 `#ABB2BF`(浅灰) |
| **中等** | **vscode-dark-plus** | 运算符使用 `#D4D4D4`(浅灰) |

## 二、修改文件

仅涉及一个文件：`frontend/src/js/cm6-syntax-highlight.js`

不修改 `hljs-themes-data.js`（自动生成文件，提示"请勿手动编辑"）。

## 三、具体修改方案

### 3.1 github-dark 主题（Line 472-542）

当前问题：大量使用浅蓝/粉紫等低饱和度粉彩色，视觉发白。

| 标记 | 旧值 | 新值 | 说明 |
|------|------|------|------|
| `function` (L486) | `#D2A8FF` (浅紫) | `#C58BFF` (中紫) | 降低亮度，提高饱和度 |
| `number` (L494) | `#79C0FF` (浅蓝) | `#58A6FF` (中蓝) | 参考 GitHub 官方真实色值 |
| `propertyName` (L491) | `#79C0FF` (浅蓝) | `#58A6FF` (中蓝) | 统一数值类颜色 |
| `string` (L495-497) | `#A5D6FF` (粉蓝) | `#79C0FF` (中蓝) | 字符串需要比普通标识符更醒目 |
| `special(tags.string)` (L496) | `#A5D6FF` | `#79C0FF` | 同上 |
| `regexp` (L497) | `#A5D6FF` | `#58A6FF` | 正则表达式颜色加深 |
| `attributeValue` (L520) | `#A5D6FF` | `#79C0FF` | 统一字符串颜色 |
| `character` (L535) | `#A5D6FF` | `#79C0FF` | 统一字符串颜色 |

### 3.2 one-dark-pro 主题（Line 396-466）

当前问题：运算符/meta/转义全部用 `#ABB2BF` 浅灰。

| 标记 | 旧值 | 新值 | 说明 |
|------|------|------|------|
| `operator` (L429-432) | `#ABB2BF` (浅灰) | `#56B6C2` (青色) | 用 One Dark 内置 hue-1 青色，易于区分 |
| `meta` (L451) | `#ABB2BF` (浅灰) | `#E5C07B` (金色) | 元信息用类型色，显眼 |
| `escape` (L458) | `#ABB2BF` (浅灰) | `#D19A66` (橙黄) | 转义用数值色，合理 |

### 3.3 vscode-dark-plus 主题（Line 244-314）

当前问题：运算符用 `#D4D4D4` 浅灰，与正文色混在一起。

| 标记 | 旧值 | 新值 | 说明 |
|------|------|------|------|
| `operator` (L277-280) | `#D4D4D4` (浅灰) | `#C586C0` (紫粉) | VS Code 实际使用的运算符颜色（TypeScript/JavaScript） |

## 四、总体原则

1. **不改变主题的视觉风格** - 仅在原有色调基础上加深，而不是换色系
2. **参考各自官方主题的真实色值** - 如 github-dark 参考 GitHub 实际 VS Code 扩展的颜色
3. **注释色不变** - 注释本应低对比度，保持不动
4. **保持运算符与关键字的区分度** - 确保运算符颜色不同于关键字和正文色

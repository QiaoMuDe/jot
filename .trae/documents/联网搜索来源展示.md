# 联网搜索来源展示

## 概述

AI 回复结束后，在消息气泡底部展示本次联网搜索的来源列表（标题、链接），用户可点击展开查看详细内容。

## 现状分析

* 搜索由 `SearchWeb()` 在后端执行，返回格式化纯文本注入 system message

* 前端搜索动画通过 `ai:search-status` 事件控制

* 搜索结果的结构化数据（`tavily.SearchAnswerResult`：Title/URL/Content/Score）当前被丢弃，仅保留格式化文本

* `TestTavilyConnection()` 依赖 `SearchWeb` 返回值判断连接是否成功

## 改动方案

### 1. `internal/services/search_service.go` — 返回结构化搜索结果

**改动**：

* 新增 `SearchSource` 结构体（Title/URL/Content）

* 新增 `SearchWebResult` 结构体（FormattedText string + Sources \[]SearchSource）

* `SearchWeb()` 返回类型从 `string` 改为 `*SearchWebResult`（失败返回 nil）

**为什么**：

* 前端需要结构化的来源数据（标题、链接、摘要）来渲染可点击的折叠面板

* 仅靠当前返回的格式化纯文本无法在前端可靠解析

### 2. `app.go` — 发射搜索来源事件

**改动**：

* 在 goroutine 内，搜索成功后（`searchResult != nil`）：

  * 将 `searchResult.Sources` 序列化为 JSON 字符串

  * 发射新事件 `ai:search-sources`，携带序列化后的 JSON

* 保持现有搜索动画事件（`searching`/`done`）不变

* 更新 `TestTavilyConnection()`：`SearchWeb` 返回 nil 时失败

**为什么**：

* 通过独立事件传递来源数据，与搜索动画解耦

* JSON 序列化确保结构数据能完整传输到前端

### 3. `frontend/src/js/ai-chat.js` — 接收来源并展示

**改动**：

* 在 `startStreaming()` 中：

  * 新增变量 `searchSources = null`

  * 添加 `'ai:search-sources'` 到 EventsOff 列表

  * 新增监听器 `unsubSources`：收到 sources JSON 后 `JSON.parse` 存入 `searchSources`

* 在 `unsubDone` 回调尾部（`scrollToBottom()` 之前）：

  * 如果 `searchSources` 不为空，在消息气泡末尾添加 `<details class="search-sources">` 折叠面板

  * 面板内以列表形式展示每条来源：序号、标题（带链接）、摘要内容

  * 默认收起状态

**为什么**：

* 来源数据在 `unsubDone` 时使用，此时 AI 回复已完成，展示时机正确

* 使用 `<details>` 折叠面板，不占用默认空间，用户按需展开查看

### 4. `frontend/src/css/components/ai-chat.css` — 来源展示样式

**改动**：

* 新增 `.search-sources` 折叠面板容器样式

* 新增 `.search-sources summary` 摘要行样式（hover accent 色）

* 新增 `.search-sources-content` 列表样式（链接、摘要排版）

* 新增 `.search-sources-content a` 链接样式

**为什么**：

* 保持整体 UI 风格一致，accent 色点缀，二级背景色区分

### 不发生变化的文件

* `ai_service.go` — AI 服务层不涉及搜索来源展示

* `types.go` — 搜索相关的类型定义放在 `search_service.go` 自身

* `index.html` — 无需新增 HTML 结构

## 数据流

```
后端 SearchWeb()
  ↓ 返回 SearchWebResult{FormattedText, Sources}
app.go goroutine
  ↓ 注入 FormattedText 到 system message
  ↓ EventsEmit("ai:search-sources", sourcesJSON)
前端 unsubSources 监听器
  ↓ JSON.parse → searchSources
AI 流式输出完成 (unsubDone)
  ↓ searchSources != null
  ↓ 创建 <details class="search-sources"> 折叠面板
  ↓ 插入到消息气泡底部、操作按钮上方
```

## 验证

1. 构建通过：`go build .` 无错误
2. 开启联网搜索提问，AI 回复结束后查看底部是否有搜索来源折叠面板
3. 点击展开，确认标题、链接、摘要显示正确
4. 关闭联网搜索时，不应出现来源面板
5. 搜索无结果时，不应出现来源面板

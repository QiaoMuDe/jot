# 新增"提示词生成"技能方案

## 一、总结

在 AI 助手的"更多技能"菜单中新增第 10 项技能"提示词生成"（`promptgen`），遵循现有技能的纯前端实现模式。

***

## 二、当前状态分析

技能系统是纯粹的**前端概念**，后端 Go 代码完全不知道技能的存在。每个技能由 4 个部分组成：

| 部分                | 位置                                | 说明                        |
| ----------------- | --------------------------------- | ------------------------- |
| HTML 菜单项          | `index.html` 第 721-768 行          | `.ai-chat-skills-item` 结构 |
| SKILL\_PROMPTS 定义 | `ai-chat.js` 第 88-293 行           | system prompt 字符串         |
| 点击处理分支            | `ai-chat.js` bindEvents() 中       | 设置 `activeSkills`         |
| chip 渲染分支         | `ai-chat.js` renderSkillChips() 中 | 显示技能标签                    |

所有非翻译技能（目前 8 项）都使用**直接字符串**类型的 prompt，互斥机制激活。

***

## 三、设计方案

### 3.1 技能行为

"提示词生成"技能激活后，AI 的角色变为**提示词工程专家**：

* **用户输入**：一句话需求描述（如"帮我写一个小红书文案生成器的 prompt"）

* **AI 输出**：一个结构完整、开箱即用的提示词，包含 Role、Task、Guidelines、Output Format 等标准章节

### 3.2 改动点

与现有非翻译技能（如"编程"、"写作"）**完全相同的模式**，4 个改动点：

**1.** **`index.html`** **— 新增菜单项**

在第 768 行（"工作总结"之后）插入：

```html
<div class="ai-chat-skills-item" data-skill="promptgen">
    <svg width="14" height="14" viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="1.5" stroke-linecap="round" stroke-linejoin="round"><path d="M14 2H6a2 2 0 0 0-2 2v16a2 2 0 0 0 2 2h12a2 2 0 0 0 2-2V8z"/><polyline points="14 2 14 8 20 8"/><line x1="16" y1="13" x2="8" y2="13"/><line x1="16" y1="17" x2="8" y2="17"/><polyline points="10 9 9 9 8 9"/></svg>
    <span>提示词生成</span>
</div>
```

图标沿用"需求规格"的文档图标（`<path d="M14 2..."`），语义匹配。

**2.** **`ai-chat.js`** **SKILL\_PROMPTS — 新增 prompt 定义**

```javascript
promptgen: `# Role: 提示词工程专家

## Core Task
根据用户的需求描述，生成一个结构完整、开箱即用的提示词（Prompt）。

## Guidelines
- 仔细理解用户的需求场景和目标
- 生成的 prompt 必须包含以下结构：
  1. **Role** — 定义 AI 的角色身份
  2. **Core Task** — 清晰描述核心任务
  3. **Guidelines** — 列出具体的执行规则和约束条件
  4. **Output Format** — （如适用）指定输出格式
- prompt 使用中文编写
- 使用 Markdown 格式，层次清晰
- 生成的 prompt 要**直接可用**，用户复制就能粘贴到任何 AI 工具中使用
- 如果用户需求不明确，可以主动追问 1-2 个关键问题来澄清
- 最终只输出 prompt 本身，不要添加额外的解释或说明

## Examples

**用户输入**: 帮我写一个翻译助手的 prompt

**输出**:
# Role: 专业翻译助手

## Core Task
将用户发送的内容翻译成指定语言。

## Guidelines
- 准确传达原文含义和语气
- 遵循地道表达，避免翻译腔
- 专业术语使用行业通用译法
- 只输出翻译结果，不添加额外内容

---

**用户输入**: 我想要一个能帮我写周报的 prompt

**输出**:
# Role: 周报撰写助手

## Core Task
根据用户提供的工作内容，生成结构化的周报。

## Guidelines
- 用 bullet points 列出本周重点工作
- 每个工作项包含：完成情况、关键成果
- 使用正式、简洁的商务语气
- 按重要性排序
- 总字数控制在 300 字以内`
```

**3.** **`ai-chat.js`** **bindEvents() — 新增点击分支**

在 skillsDropdown 的 click 委托中，在 `report` 分支后新增：

```javascript
case 'promptgen':
    activeSkills = {};
    activeSkills.promptgen = true;
    renderSkillChips();
    closeSkillMenu();
    break;
```

与 coding/writing/tutor 等 8 项技能完全相同的处理逻辑。

**4.** **`ai-chat.js`** **renderSkillChips() — 新增渲染分支**

```javascript
case 'promptgen':
    label = '提示词生成';
    break;
```

## 五、验证步骤

1. 打开 AI 助手 → 点击"更多技能" → 确认"提示词生成"出现在菜单中
2. 点击"提示词生成" → 确认菜单关闭、输入框上方显示对应 chip
3. 发送消息"帮我写一个小红书文案生成器的 prompt" → 确认 AI 按提示词工程专家角色回复结构化的 prompt
4. 点击 chip 上的 × 取消技能 → 确认恢复正常对话
5. 切换会话 → 确认技能自动重置

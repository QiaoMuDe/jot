# 上下文大小显示组件设计方案

## 一、总结

在 AI 助手对话界面增加一个上下文 token 用量显示组件，采用前端字符估算方案，放置在头部标题栏右侧的 `view-controls` 容器中。

***

## 二、当前状态分析

### AI 聊天视图结构

```
#viewAiChat
  .view-header
    button#aiChatBackBtn     ← 返回按钮
    h2#aiChatTitle           ← "AI 助手" 标题
    div.view-controls        ← 【空容器】当前没有任何内容
  .ai-chat-layout
    .ai-session-sidebar      ← 左侧会话侧栏（可折叠）
    .ai-chat-content         ← 右侧对话区
      #aiChatMessages        ← 消息列表
      #aiChatInputArea       ← 底部输入区
        .ai-chat-toolbar     ← 工具栏（模型选择、深度思考、技能）
        .ai-chat-input-row   ← 输入框 + 发送按钮
```

### 关键发现

1. **`div.view-controls`** **是空的预留容器** — 零成本利用，不需改布局
2. 项目使用纯 CSS 变量主题系统（12 套主题），组件需通过 `var(--xxx)` 适配所有主题
3. 前端状态 `chatHistory` 已在内存中，可直接遍历估算 token
4. 现有 CSS 设计语言：温暖极简、浅色为主、圆角柔和、低阴影

***

## 三、设计方案

### 3.1 放置位置：头部 `.view-controls` 容器 ✅

选择理由：

| 维度         | 评价                               |
| ---------- | -------------------------------- |
| **可见性**    | 始终可见（不像侧栏会折叠），用户打开 AI 助手第一眼就能看到  |
| **空间零成本**  | `view-controls` 已是空容器，直接填充，无需改布局 |
| **干扰最小**   | 在对话区之外，不影响消息列表和输入区的核心交互          |
| **语义匹配**   | 头部状态栏是最自然的状态信息展示位置               |
| **其他选项对比** | 工具栏已过挤（4 个控件），侧栏底部不显眼，输入区底部冲突    |

### 3.2 视觉设计

采用**纯文字标签**形式，极致简洁：

```
┌──────────────────────────────────────────────┐
│  ← 返回      AI 助手              12.4K tokens │
└──────────────────────────────────────────────┘
```

**设计要点：**

1. **布局**：`.view-controls` 使用 `display:flex; align-items:center; margin-left:auto` 右对齐
2. **文字**：`font-size: 0.75rem`，颜色 `var(--text-muted)`，格式 `12.4K tokens`
3. **交互**：仅显示，无悬停/点击交互

### 3.3 交互行为

| 场景           | 行为                                    |
| ------------ | ------------------------------------- |
| **无消息（空对话）** | 不显示（`display:none`）                   |
| **有消息**      | 显示 `X.XK tokens`                      |
| **切换会话**     | 重新计算新会话的 token 数                      |
| **发送/接收消息**  | 实时更新（每次消息变更后调用 `updateContextSize()`） |

### 3.4 token 估算函数

```javascript
function estimateTokens(text) {
  if (!text) return 0;
  const chineseChars = (text.match(/[\u4e00-\u9fff]/g) || []).length;
  const otherChars = text.length - chineseChars;
  return Math.ceil(chineseChars / 1.5 + otherChars / 4);
}

function calcContextTokens(chatHistory) {
  // 对话历史
  let total = chatHistory.reduce((sum, msg) => sum + estimateTokens(msg.content), 0);
  
  // 笔记引用上下文
  if (cachedRefContext) {
    total += estimateTokens(cachedRefContext);
  }
  
  // 技能 system prompt
  const activeSkillId = Object.keys(activeSkills)[0];
  if (activeSkillId && SKILL_PROMPTS[activeSkillId]) {
    const skillConfig = activeSkills[activeSkillId];
    const skillKey = typeof skillConfig === 'object' ? skillConfig.dir || skillConfig.mode || activeSkillId : activeSkillId;
    const promptText = SKILL_PROMPTS[activeSkillId][skillKey] || SKILL_PROMPTS[activeSkillId];
    total += estimateTokens(promptText);
  }
  
  return total;
}
```

## 五、假设与决策

| 决策       | 选择                  | 理由               |
| -------- | ------------------- | ---------------- |
| 放置位置     | 头部 `.view-controls` | 现成空容器，始终可见，零布局影响 |
| token 估算 | 前端字符估算              | 够用、零依赖、速度快       |
| 显示内容     | 仅当前 token 数（无上限）    | 少即是多，用户不需要参考值    |
| 显示格式     | `X.XK tokens`       | 简洁统一             |

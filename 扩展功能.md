# 可以扩展的方向（按价值/成本排序）

### 高价值 · 中低成本 — 最推荐的 3 个

| 功能                     | 说明                                                         | 原理                                                         | 价值                                  | 成本                             |
| ------------------------ | ------------------------------------------------------------ | ------------------------------------------------------------ | ------------------------------------- | -------------------------------- |
| **Markdown 快捷操作**    | Ctrl+B 加粗（`**粗体**`）、Ctrl+I 斜体（`*斜体*`）、Ctrl+U 下划线、Ctrl+K 插入链接 | CM6 keymap 扩展，选中文本后包裹 Markdown 符号                | 极高 — 写作最频繁的操作，不用手打符号 | 低 — ~50 行 JS                   |
| **格式化工具栏**         | 编辑器顶部出现一行工具按钮：**B** / *I* / H1-H3 / 链接 / 列表 / 引用 / 代码 | 调用 CM6 `EditorView.dispatch` 插入或包裹语法                | 高 — 对非 MD 用户友好，降低学习门槛   | 中 — ~200 行 JS+CSS              |
| **粘贴图片**             | 剪贴板截图直接 Ctrl+V 粘贴到笔记中，自动保存为本地文件并插入 `![]()` | 监听 paste 事件，base64 → 文件存储 → 替换为 markdown 图片链接 | 高 — 截图党刚需                       | 中 — 需前端读剪贴板 + 后端存文件 |
| **大纲/目录（Outline）** | 右侧或侧栏显示文档标题层级导航，点击跳转                     | 中 — 长文档场景                                              | 低 — 从 CM6 语法树提取 heading 节点   |                                  |

# 待办保存动画优化方案

## 当前问题

编辑待办保存后，`finishEdit()` 调用 `loadTodos()` 触发**全量重新渲染**（`renderTodos` 重建整个 `listEl.innerHTML`），所有条目重新生成 DOM，即使内容没变也会播放入场动画（`.todo-enter` 的 0.2s 淡入），视觉上就像整个列表"闪烁"了一下。

## 设计方案

### 核心理念

保存后不重新渲染整个列表，改为**原地更新 + 保存成功动画**。动画设计采用优雅的"涟漪确认"效果——从左下角主题色竖线开始，一道柔和的光晕拂过条目，配合微妙的"呼吸"缩放，给人"保存成功"的确定感。

### 文件变更

#### 1. `frontend\src\js\...\todo.css` — 新增保存动画

添加 `@keyframes todoSaveSuccess` 和 `.todo-item.todo-saved` 类：

- **动画效果**（6 个阶段，总时长 0.7s）：
  - 0% → 15%：背景从卡片色过渡为 `var(--accent)` + 10% 透明度的淡色叠加，左侧 `border-left` 从 `--accent` 变为 `--accent` 亮色变体（或维持不变但加亮），同时 `box-shadow` 增强
  - 15% → 50%：卡片微幅"呼吸"膨胀（`scale(1.01)`），背景色逐渐淡出
  - 50% → 85%：恢复原始尺寸（`scale(1)`），背景色完全恢复
  - 85% → 100%：轻微的阴影尾迹消失
- 整体动画曲线使用 `cubic-bezier(0.34, 1.56, 0.64, 1)` 弹性曲线，配合柔和的缓出

#### 2. `frontend\src\main.js` — 修改 `editTodo()` / `finishEdit()`

- **保存分支**（`save = true`）：
  - 调用 `window.go.main.App.UpdateTodo(id, newText)` 保存到后端
  - **不再调用 `loadTodos()`**
  - 原地将 textarea 替换回 span，更新 span 的文本内容
  - 给 `item` 添加 `.todo-saved` 类触发动画
  - 在 `animationend` 事件（或 700ms setTimeout）后移除 `.todo-saved` 类
  - 调用 `refreshTodoStats()` 更新统计信息
  - 发送 `'todos-updated'` 事件供其他模块同步

- **取消分支**（`save = false`）：
  - 保持现有行为：恢复原始文本，移除 `.editing` 类，不做任何动画
  - **也不调用 `loadTodos()`**

### 设计细节

- **色调**：使用 `var(--accent)` 作为动画主色，保持与品牌一致，不引入突兀的新颜色
- **时间**：0.7s 的完整动画周期——足够让用户感知到"保存完成"，又不会显得拖沓
- **触发时机**：只在**真正有内容变更**时播放动画（`newText !== oldText`）。如果内容没有变化，直接静默恢复，不播放动画
- **边界情况**：如果动画播放期间用户再次双击进入编辑，`.todo-saved` 会被 `editing` 类覆盖，不会冲突

### 当前状态需要保留的行为

- Enter 键保存、Escape 取消、Ctrl+Enter 换行 —— 保持不变
- 失焦自动保存 —— 保持不变
- 编辑时隐藏复选框和删除按钮 —— 保持不变

## 验证方式

1. 双击编辑一个待办，修改内容后按 Enter
2. 应该看到条目有一个优雅的"涟漪"光晕动画，列表其他条目不受影响
3. 整个列表不会闪烁或重新渲染
4. 编辑后内容正确，统计信息更新
5. 不修改内容直接保存（Enter），应该没有任何动画，静默退出编辑

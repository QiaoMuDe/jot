# 统一 AI 会话侧栏菜单 Spec

## 为什么改

当前 AI 会话侧栏中，**右击** 弹出独立菜单（重命名 + 导出），**更多按钮（⋯）** 弹出另一个菜单（置顶 + 重命名 + 删除会话）。两个菜单功能割裂，用户需在不同位置寻找不同操作，体验不一致。

## 变更内容

- 将两个菜单合并为一个统一菜单，包含所有操作项：**置顶/取消置顶**、**重命名**、**导出**、**删除会话**
- **右击** 和 **更多按钮点击** 弹出完全相同的菜单（位置不同：右击在鼠标位置，更多按钮定位在按钮旁）
- 移除 HTML 中硬编码的 `#aiSessionContextMenu` 元素
- 移除 `ai-session-context-menu` 相关 CSS 类
- 右击复用的下拉菜单动态构建逻辑，用 `.ai-session-more-menu` 样式

## 影响范围

- 受影响的 spec：AI 助手模块（会话管理）
- 受影响的代码：
  - `frontend/index.html` — 移除 `#aiSessionContextMenu`
  - `frontend/src/js/ai-chat.js` — 修改右击/更多菜单逻辑
  - `frontend/src/css/components/ai-chat.css` — 移除旧右键菜单样式

## 新增需求

### 需求：统一会话菜单

系统应提供一个统一菜单，包含所有会话操作项。

#### 场景：右击弹出统一菜单
- **当** 用户在 AI 会话条目上右击
- **则** 在鼠标位置弹出统一菜单，包含置顶/取消置顶、重命名、导出、删除会话四项

#### 场景：更多按钮点击弹出统一菜单
- **当** 用户点击 AI 会话条目右侧的 ⋯ 按钮
- **则** 在该按钮旁弹出统一菜单，内容与右击菜单完全一致

#### 场景：菜单操作
- **当** 用户点击"置顶/取消置顶" → 切换置顶状态，刷新列表
- **当** 用户点击"重命名" → 关闭菜单，触发内联编辑
- **当** 用户点击"导出" → 执行导出，显示结果通知
- **当** 用户点击"删除会话" → 弹出确认框，确认后删除

## 修改的需求

### 需求：移除旧的右键菜单
**变更**：删除 `index.html` 中 `#aiSessionContextMenu` 元素及其内容，删除 `ai-chat.css` 中 `.ai-session-context-menu` 及相关样式，删除 `ai-chat.js` 中的 `sessionContextMenu` 引用和 `showSessionContextMenu`/`closeSessionContextMenu` 函数。

## 删除的需求

### 需求：独立的右键菜单（#aiSessionContextMenu）
**原因**：被统一菜单替代
**迁移**：右击改用动态构建的统一菜单（`.ai-session-more-menu`）
